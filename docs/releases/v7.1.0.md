# v7.1.0

Release date: 2026-07-04

## Summary

This minor release improves feature consistency by introducing a formal feature-handling standard and applying it through a Build Counts extraction pilot, while refining Map Art material-count behavior.

## Highlights

- Added a Feature Handling Standard to keep feature structure and ownership consistent.
- Extracted Build Counts into dedicated core/UI feature modules and integrated them into studio flow.
- Excluded Map Art default black background from required color counts.
- Added targeted counts-core unit coverage and preserved full end-to-end/a11y validation.

## Included Updates

- Added `docs/feature-handling-standard.md` and linked it from architecture and README docs.
- Added `features/counts-core/count-rules.js` and `features/counts-core/count-rules.test.js`.
- Added `features/counts-ui/color-usage-ui.js` and wired it from `features/studio-ui/studio-ui.js`.
- Added counts feature script imports in `index.html`.
- Updated Map Art count behavior and E2E assertions in `features/studio-e2e/studio-e2e.test.cjs`.
- Updated release metadata in `package.json`, `index.html`, `CHANGELOG.md`, `README.md`, and `docs/releases/README.md`.

## Accessibility Compliance

- Standard target: WCAG 2.2.
- Default E2E pipeline still includes WCAG 2.2 AA checks.

## Validation

- `npm run test` passed (core + e2e + WCAG 2.2 AA).
