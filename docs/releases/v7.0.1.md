# v7.0.1

Release date: 2026-07-04

## Summary

This patch release tightens Map Art mode behavior by making it explicitly fixed at 128x128, restoring non-map size preferences when Map Art is disabled, and setting black as the default Map Art background.

## Highlights

- Enabling `Map Art View` now forces the canvas to `128x128`.
- Disabling `Map Art View` restores the user's stored non-map canvas size.
- Map Art mode no longer overwrites stored non-map size preferences.
- Map Art default/clear background is now black.

## Included Updates

- Updated Map Art toggle and resize persistence behavior in `features/studio-ui/studio-ui.js`.
- Updated canvas size inputs to support up to `128` in `index.html`.
- Expanded Map Art E2E regression coverage in `features/studio-e2e/studio-e2e.test.cjs`.
- Updated release metadata in `package.json`, `index.html`, `CHANGELOG.md`, `README.md`, and `docs/releases/README.md`.

## Accessibility Compliance

- Standard target: WCAG 2.2.
- Default E2E pipeline still includes WCAG 2.2 AA checks.
