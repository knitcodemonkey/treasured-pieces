# v7.2.0

Release date: 2026-07-04

## Summary

This minor release introduces a dedicated motif system for Map Art View, separating map-scale motifs from standard studio motifs so 128x128 workflows get appropriately large pattern options.

## Highlights

- Split motif libraries into standard and map-art sets.
- Added oversized map-art motifs specifically for 128x128 map canvases.
- Updated studio motif selection and loading to switch libraries automatically with Map Art View.
- Preserved passing core, e2e, and accessibility validation.

## Included Updates

- Added `features/project-core/motifs/map-art-single-motifs.js`.
- Added `features/project-core/motifs/map-art-border-motifs.js`.
- Updated `features/project-core/motifs/index.js` to export both standard and map-art template definitions.
- Updated `features/project-core/project-core.js` to build and expose `project.mapArtTemplates` and support library-aware motif application.
- Updated `features/studio-ui/studio-ui.js` so motif picker content and motif load behavior are mode-aware (`standard` vs `mapArt`).
- Added/updated map-art motif separation assertions in `features/project-core/project-core.test.js`.
- Updated release metadata in `package.json`, `index.html`, `CHANGELOG.md`, `README.md`, and `docs/releases/README.md`.

## Accessibility Compliance

- Standard target: WCAG 2.2.
- Default E2E pipeline continues to include WCAG 2.2 AA checks.

## Validation

- `npm run test` passed (core + e2e + WCAG 2.2 AA).
