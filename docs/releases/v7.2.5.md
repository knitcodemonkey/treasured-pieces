# v7.2.5

Release date: 2026-07-04

## Summary

This patch release publishes CI reliability hotfixes that remove runtime Playwright browser installation from checks and prevent install-step stalls.

## Highlights

- Moved E2E and accessibility jobs to the official Playwright container image.
- Removed runtime `npx playwright install --with-deps chromium` from CI jobs.
- Ensured jobs use preinstalled browsers via `PLAYWRIGHT_BROWSERS_PATH=/ms-playwright`.
- Kept existing validation gates intact while reducing setup volatility.

## Included Updates

- Updated `.github/workflows/e2e.yml` to run inside `mcr.microsoft.com/playwright:v1.54.2-jammy` and skip runtime browser installation.
- Updated `.github/workflows/accessibility.yml` to use the same prebuilt Playwright runtime strategy.
- Updated release metadata in `package.json`, `package-lock.json`, `README.md`, `CHANGELOG.md`, `docs/releases/README.md`, and `index.html`.

## Accessibility Compliance

- Standard target: WCAG 2.2.
- Default E2E pipeline continues to include WCAG 2.2 AA checks.

## Validation

- `npm run test` passed (core + e2e + WCAG 2.2 AA).
