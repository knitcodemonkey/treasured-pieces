# v6.1.22

Release date: 2026-07-04

## Summary

This patch release improves motif loading and canvas continuity with guided resize prompts and persisted canvas dimensions, and strengthens quality gates by expanding end-to-end coverage and making accessibility checks part of the default E2E pipeline.

## Highlights

- Added size-aware motif loading with an expansion prompt when a selected motif is larger than the active canvas.
- Persisted canvas dimensions in local storage so the previous size is restored after refresh or return.
- Added explicit E2E coverage for recent UX behaviors and included WCAG 2.2 AA scanning in default E2E runs.

## Included Updates

- Added motif source size metadata in `features/project-core/project-core.js`.
- Added oversized motif resize confirmation and shared resize persistence flow in `features/studio-ui/studio-ui.js`.
- Added E2E assertions for counts accordion toggling, zero-count filtering, oversize motif cancel/accept paths, and persisted size restore in `features/studio-e2e/studio-e2e.test.cjs`.
- Added WCAG 2.2 AA accessibility checks to the default E2E script in `features/studio-e2e/run-e2e.sh`.
- Updated release metadata in `package.json`, `index.html`, `CHANGELOG.md`, `README.md`, and `docs/releases/README.md`.

## Accessibility Compliance

- Standard target: WCAG 2.2.
- E2E now includes `wcag22aa` accessibility scans by default via axe.

- `npm run test` passed.
