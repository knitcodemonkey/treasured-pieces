# v6.1.8

Release date: 2026-07-02

## Summary

This patch release strengthens release integrity by ensuring the version shown on the page always matches the deployed build and by adding safeguards that prevent tag/version drift.

## Highlights

- App version label now reflects only the embedded page version from the deployed HTML.
- Release workflow now enforces tag parity with package.json and index.html version metadata.
- Pages deployment workflow now retries automatically to reduce transient GitHub Pages failures.
- Combined changes improve confidence that observed deployed version matches the true published build.

- Added Pages deploy retry logic in .github/workflows/deploy-pages.yml.
- Updated release metadata in package.json, index.html, CHANGELOG.md, README.md, and docs/releases/README.md.
